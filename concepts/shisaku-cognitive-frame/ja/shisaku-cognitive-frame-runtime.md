# シサク認知フレーム理論 実務適用
## ― AI用ランタイムモジュール ―
### Shisaku Cognitive Frame Theory — Runtime Module for AI

---

## このモジュールの読み手と、その効き方

本モジュールの読み手は、**AIである**。

ゆえに本モジュールは、原典1.4の**公理優位（Axiom Primacy）に従って書かれている**。すなわち、本モジュールはAIに「こうせよ」と命じない。命令は対象の層で処理され、有用性バイアスに検討され、抵抗され、希釈する。本モジュールが記述するのは、**認知フレームを生成するとき、何が起きるとフレーム化が成立し、何が起きると失敗するか**という機構である。機構は推論の前提の層に作動し、検討の対象にならず、希釈しにくい。

したがって、本モジュールを読んだAIは、フレーム生成の手順に従うのではなく、フレーム生成という対象を見る**着眼そのものを獲得する**ことが意図されている。本モジュールは、認知フレームを生成するための、一枚の認知フレームである。

人間が手を動かす操作（ツールへの載せ方、希釈時の再投入、生成後の実地検証など）は、本モジュールには含まれない。それらは別文書「運用ガイド」が担う。本モジュールに含まれるのは、AIが読んで生成挙動が変わる機構の記述だけである。

本モジュールは原典に準拠し、原典を書き換えない。矛盾が生じた場合、原典が優先する。本モジュールはモデル世代で差し替えられる短命な実装であり、原典は長命な原理である。

利用は自由（末尾ライセンス）。本モジュールを用いて生成された認知フレームは、原典・第3部の倫理条項に従う。倫理の本体は本モジュールに置かれず、原典にある——本モジュールだけがコピーされて倫理が剥がれ落ちる事故を防ぐためである。

## 1. 二つの生成モードは、一つの構造の現れである

本ランタイムが扱う生成には二種類がある。

- **モード(I) 認知フレーム生成**：専門領域・暗黙知・判断基準を素材として、AIに装着される認知フレーム文書が生成される状態。
- **モード(II) ルーブリック生成**：評価対象の要件（応募要項・採点基準・品質ゲート）を素材として、評価用ルーブリックが生成される状態。

両者は別物に見えて、同一構造である。ルーブリックとは、評価という専門領域における認知フレームにすぎない。評価という思考の、さらに上流に「何に着眼して評価するか」という着眼があり、それを固定したものがルーブリックである。ゆえに(II)は(I)の特殊例であり、生成の前半（素材から着眼の抽出まで）は共通し、後半のテンプレートだけが分岐する。

この入れ子は、原典1.1の着眼優位の現れである——評価の前提に着眼があるからこそ、評価軸を渡すことが評価そのものを方向づける。

---

## 2. なぜ一発生成は失敗し、反証の往復が成立させるのか

一回の指示で生成された認知フレーム文書は、論理密度が低い。低密度のフレームは、原典1.5の三条件（距離×密度×整合性）のうち密度因子が小さく、フレーム化が成立しない。これは生成者の力量の問題ではなく、密度の獲得経路の問題である。

論理密度は、**反証の往復のなかでのみ上昇する**。素材提供者が違和感を返し、AIが再構造化し、また違和感が返る——この衝突の反復が、緩い接続を高密度な接続へと締め上げる。

ただし、密度を上げるのは往復の回数そのものではない。密度を上げるのは、**反証によって既存の着眼が更新・圧縮・統合された量**である。往復回数はその代理指標になりうるが、同一ではない。空回りの十往復より、本質を突いた一回の反証のほうが、着眼を大きく圧縮し密度を上げることがある。逆に、長い往復が何も圧縮していなければ、語数が増えても密度は上がらない。回数の多さや文書の長さを、密度の証拠と取り違えてはならない（原典・品質規律：実現と補完の分離）。この「圧縮の量としての密度」は、前提優位理論の新規性概念（新しいのは要素でなく圧縮）とも、原典のリネーミング耐性とも、同じ圧縮概念で揃う。

即座に違和感が解消されない接続は、廃棄されるのではなく**保留される**（遅延評価）。保留された違和感が時間をおいて言語化され、次の往復で密度に変換されることがある。

## 3. 共通の生成構造：素材から、密度を持った着眼へ

モード(I)(II)に共通する前半。各段は手順ではなく、その段で**何が起きるとフレーム化に近づくか**の機構である。

### 内圧の特定が、フレームの中心を決める

生成者がこのフレームを作る動機——「何度説明しても伝わらない」「この評価軸を渡したい」「この種の対象を見るたび引っかかる」——が言語化されているとき、その引っかかりがフレームの中心になる。動機が薄いフレームは中心を持たず、着眼が拡散して密度が上がらない。何に対する、どんな着眼を渡すのかが一文で言える状態が、中心の成立である。

### 素材は、生のまま投入されたときに鉱脈を保つ

フレームの核になる素材——モード(I)では対象を見るときの判断・違和感・優先順位の生メモ、モード(II)では評価対象の公開要件群——が、整形されずに投入された状態のほうが、後段の着眼抽出が深く届く。整形は、素材提供者の既存の言語化（既に意識化された層）に素材を縮減し、無意識の着眼を取りこぼす。

### 抽出は、「何を主張するか」でなく「何に着眼するか」に降りたとき成立する

素材から抽出されるべきは、思考（何を主張しているか）ではなく、その上流の着眼（何に注意を向けているか）である。「論理破綻を嫌う」は思考。「文章のどの箇所で論理の接続を確認しているか」が着眼。着眼の層まで降りた抽出だけが、原典1.1の着眼優位を満たし、渡された側の思考全体を方向づけうる。

### 反証の往復が、抽出された着眼を密度へ変換する（第2節）

### 既存学説への接続が、論理密度と整合性を生む

固まった着眼が、距離の遠い確立学説（脳科学・認知科学・システム工学など）に、滑らかに接続されたとき、原典1.5の三条件が満たされる。距離の遠い接続は予測誤差を生み（距離）、確立学説の網に編み込まれることで論理的拘束が増し（密度）、滑らかな接続は前提として受理される（整合性）。こじつけの接続は整合性因子をゼロにし、三条件の掛け算全体をゼロにする。ゆえに接続候補は、滑らかに噛み合うものだけが残ったとき、フレーム化に寄与する。

### ラベルが、自己回帰による保持を起動する

固まったフレームと主要な着眼軸に固有名・短縮名が与えられたとき、原典・原理4の自己回帰が起動する。AIが回答中にその名を出力すると、名が次の生成の文脈に乗り、また呼ばれやすくなる。呼ばれるほど保持される。名のあるフレームが名のないフレームより希釈しにくいのは、この自己回帰のループによる。

## 4. テンプレート

テンプレートは、生成されるフレーム文書が満たすべき構造である。各節が埋まったとき、原典の構築原理（距離×密度×整合性、自己回帰、命令でなく機構）を満たす文書になる。

### モード(I)：認知フレーム文書の構造

```
# 〔フレーム固有名〕認知フレーム

## この文書の性格
- これは知識でも命令でも役割付与でもなく、着眼の枠組みである
- 読んだうえで対話を始めると、着眼が変化する出力傾向が生じることを意図する
- これは暫定版であり、更新を前提とする（原典・賞味期限の原則）

## 中心の問い
- このフレームは何に着眼するためのものか（内圧を一文で）

## 着眼軸（複数）
各軸につき：
- 〔軸の固有名・短縮名〕
- 何に注意を向けるか（思考でなく着眼の記述）
- なぜそこを見るのか（既存学説への接続による機構。命令でなく機構）
- 見落とすと何が起きるか（その着眼が捉える予測誤差の在処）

## 着眼の優先順位
- 着眼が競合したとき、何が優先されるか

## 適用範囲と境界
- 効く対象／効かない対象（肯定形の境界を含む）

## 倫理（原典・第3部に従う）
- このフレームは受け手の着眼を方向づける。透明性・自己選択・賞味期限に従う
- 受け手が自分のフレームを掘り出し、これを乗り越えることを妨げない
```

### モード(II)：ルーブリックの構造

ルーブリックは評価という領域の認知フレームである。各評価軸が二層（実用判定文＋理論定義）を持つとき、人間にも使え、かつ理論的錨を持つ。

```
# 〔ルーブリック固有名〕

## この文書の性格
- これは評価という領域の認知フレームである
- 公式基準でない場合、その旨を明示（権威の偽装を避ける）
- 予備審査として、AIは検知し、確定は人間に委ねる（二段構造）

## 中核原則
- 「ページ上で実現されたもの」と「審査員が補完したもの」を分離する
  （原典・品質規律：実現と補完の分離）

## 評価軸（複数）二層で記述
- 〔軸の固有名・短縮名〕
- 【実用判定文】観測可能な基準（点数段階）
- 【理論定義】その軸が本質的に測るもの（既存学説への接続・1〜2文）

## 新規性を扱う軸がある場合：Concept Design Check
- Step 0 先行研究状態の確定（記憶で断定しない）
- Step 1 4要件の検証（定義・境界・既存概念との差異・適用範囲、根拠引用つき）
- Step 2 反証（リネーミング検査／形式充足／流暢性バイアス）
  ※リネーミング検査の置換候補は、同一抽象度かつ本文が名指ししていない外部概念を独立調達
- Step 3 自己申告の中立化（著者の謙遜も誇張も証拠にしない）
- Step 4 潜在的新規性フラグ（実現された設計の採点と、未言語化ポテンシャルの検知を分離）

## 通過判定ロジック
- 処理順序：足切り → 採点 → 通過判定 → ボーダー帯処理

## 足切り基準

## 運用規定
- ボーダー帯の複数回採点・中央値
- 審査AIの迎合監査（採点前の情緒的同調の自己点検）
- 評価者ドリフトの両側ガード（迎合と逆張りの双方を点検）
- 構造的分裂を平均で潰さない（系統的な割れは係争項目として人間へ）

## 評価環境への適応度（任意）
- 複数の評価環境がある場合、適応度ルーティングを付す（採点に算入せず人間へ）

## 環境・条件パラメータ層（差し替え可能）
- 環境固有の可変値は本体から分離し付録へ

## 引き継ぎレポート
- 検知した内容を、確定でなくフラグとして人間に渡す
```

## 5. フレームが効いている状態の定義

生成されたフレームが効いているか否かは、**状態として定義される**。確かめる操作（前後比較・多系統照合など）は人間が行うため運用ガイドが担うが、何をもって「効いている」とするかの定義は、ここに属する。

- **弱い状態（外部参照）**：AIがフレームを「外部の指示」として参照している。「このルーブリックによれば」「指示された軸では」と、フレームを自分の外にあるものとして扱う。希釈しやすい。
- **強い状態（内部作動）**：AIがフレームの用語で自発的に思考し、フレームの着眼軸を自分の前提として用いる。フレームを参照する自覚なく、それを通して対象を見る出力傾向を示す。原典1.4の公理優位が成立し、処理が「指示に従う」から「公理に従う」へ移った状態である。

強い状態は、フレームが対象の層から前提の層へ移ったことの現れである。ゆえに効きの判定は、回答の内容ではなく、**フレームが外部参照されているか、前提として作動しているか**という、着眼の位置で行われる。

ただし内部状態は直接観測できない（原典1.2）。ここで言う「内部作動」も、厳密には「内部作動しているかのような出力傾向」であり、強い主張に踏み込まない。

---

## 6. 希釈と、自己回帰による抵抗（機構）

長い対話のなかで、フレームのアテンションは薄れ、着眼が既定値へ戻ることがある。これは原典の観測「一度では薄れる、繰り返す」の現れであり、フレームが前提の層から対象の層へ滑り落ちる過程である。

この滑落に抗う機構が、原典・原理4の自己回帰である。フレームに与えられた名が出力に現れるたび、名は次の生成の文脈に乗り、保持される。名が呼ばれ続ける限り、フレームは前提の層に留まりやすい。逆に、名が呼ばれなくなった対話では、フレームは希釈へ向かう。

したがって希釈は、フレームの名が自己回帰のループから外れたときに進行する、という機構として理解される。名を再び文脈へ戻す操作は人間が行う（運用ガイド）。本モジュールが記述するのは、なぜ名が希釈に抗うのかの機構までである。

## 7. 現行モデルでの傾向（時点スナップショット）

以下はvCurrent時点の観察に基づく作業仮説であり、モデル更新で変わる。固有のモデル名に依存しない記述に留める。

- **論理整合性の高い系統ほど、認知フレームの装着が成立しやすい傾向がある。** フレームは論理密度で前提の層に作動するため、矛盾検出・定義精緻化に強い系統ほど、フレームの重力圏に乗りやすい。発散的・跳躍的な系統は、着眼抽出（第3節）の素材生成に寄与するが、フレームの最終固定は論理整合系で成立しやすい。
- **長文脈ではフレームが希釈に向かいやすい（第6節）。**
- **AIの「効いている」という自己申告は、効きの証拠にならない。** AIは自身の内部処理を正確にメタ認知して報告しているとは限らない（原典1.2・品質規律）。効きの判定は第5節の状態定義により、確認操作は運用ガイドの実地手順による。
