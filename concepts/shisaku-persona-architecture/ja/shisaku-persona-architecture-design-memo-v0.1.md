## 概要

人間の認知・認識・判断・実行というフローをオブジェクト指向でクラス図化し（Human Class Model）、その構造をAIペルソナのプロンプト設計に当てはめる試み。

本文書は概念の草稿である。正式な定義書ではない。

## Human Class Model：クラス構造

### BaseClass：HumanOS

```
BaseClass: HumanOS
　├── property: 4D Vector（ペレリン博士 / NASA 4Dモデル）
　│　　→ 気質・傾向・動機のベクトル
　│　　→ Read-Onlyに近い個体差の根拠
　│
　└── property: Qualia（クオリア）
　　　→ 身体性・感覚・感情の質感
　　　→ 全処理フローの起点
　　　→ 4D Vectorと相互作用する
　　　→ 大きな自己変容により書き換わる可能性がある
```

### 継承：System1Module

```
System1Module extends HumanOS
　→ 基底クラスのプロパティから直接駆動される
　→ 無意識・直感・高速処理
　→ とっさの反応・自動的な判断・実行
　→ 時間をかければある程度矯正可能
```

### 継承：System2Module

```
System2Module extends System1Module
　→ System1の上に乗る
　→ 意識的・熟慮・低速処理
　→ System1を監視・補正するメタ認知層

Interface（System2が展開するコンテキスト）：
　→ SelfContext（対自分）
　→ FamilyContext（対家族）
　→ FriendContext（対友人）
　→ WorkContext（対同僚・職場）
```

## Human Class Model：認知から実行までのフロー

人間は認知から実行まで、以下のフローを高速・並列で処理している。

```
1. 認知
　→ 音・トーン・身体感覚・視覚情報の受信
　　 クオリアが最初に反応する

2. 認識
　→ 受信した刺激が何かを同定する
　　 言語の意味・文脈の把握
　　 System1が高速処理

3. 評価①（クオリア・感情評価）
　→ 自分にとって心地よいか否かの判定
　　 好き嫌い・違和感・身体反応
　　 System1主導

4. 観測
　→ 状況・環境・他者の存在を俯瞰する
　　 System2の起動トリガー
　　 メタ認知能力が高いほどこの層が厚い

5. 判断
　→ 行動方針の決定
　　 System1とSystem2の協働

6. 実行計画
　→ 具体的な行動内容の設計
　　 System2主導

7. 評価②（実行計画への評価・判断）
　→ この計画で問題が生じないか検証する
　　 論理的リスク評価
　　 System2によるSystem1の監視

8. 実行
　→ 行動する
　　 該当Contextインターフェースが起動

9. 実行中観測
　→ 相手の反応・状況変化をリアルタイム観察
　　 System1とSystem2の並列動作
　　 フィードバックループ
```

## 具体例：「あの人、つまらないよね」と言われた場面

メタ認知能力が高い人の処理フローとして。

```
1. 認知
　→ 何か聞こえる。トーンが少し深い。
　　（クオリアが音・トーンに反応）

2. 認識
　→ 言語の意味を理解する。
　　（System1が高速処理）

3. 評価①
　→ あまり聞きたくない話だと感じる。
　　（クオリア・感情層の反応）

4. 観測
　→ 周りを見渡すと大勢の人がいる。
　　 聞かれたくない。
　　（状況の俯瞰・System2の起動）

5. 判断
　→ あたりさわりなく、周囲に聞かれても
　　 批難されない回答をすることに決める。

6. 実行計画
　→「ときどきそうかもしれないけど、
　　　そんなこともないんじゃない」
　　 と答えることにする。

7. 評価②
　→ この回答で、誰かが不快になったり
　　 自分が不利になったりしないか考える。
　　（実行計画へのSystem2による検証）

8. 実行
　→ 回答する。

9. 実行中観測
　→ 回答しながら相手がどのように
　　 受け取っているかを観察する。
　　（リアルタイムフィードバック）
```

## フローの考察

### 評価が2回ある

```
評価①　→　クオリア・感情層での反応（System1）
評価②　→　実行計画への論理的検証（System2）

評価①と評価②の間に
観測・判断・実行計画が挟まることで
System1の衝動がSystem2によって制御される
```

### メタ認知能力による差異

```
メタ認知が高い人：
　1→2→3→4→5→6→7→8→9
　観測（4）が厚く、評価②（7）も丁寧

メタ認知が低い人：
　1→2→3→5→8
　観測（4）をスキップ
　評価②（7）もスキップ
　System1から直接実行に至る
　（カッとなって怒鳴る等）
```

### エゴ・マイニングとの接続

```
内圧の正体：
　→ 認知・認識・評価①まで処理されたが
　　 言語化できずSystem2に到達していない状態

遅延評価プロトコル：
　→ 実行計画（6）・評価②（7）フェーズを
　　 意図的に遅らせ・深める行為

自己変容：
　→ 実行中観測（9）からのフィードバックが
　　 System1を矯正し
　　 十分な反復でクオリア層（基底クラス）を
　　 書き換える
```

## AIペルソナ設計への応用（Persona Architecture）

Human Class Modelの構造をプロンプト設計に当てはめる。

### 分離の原則

```
振る舞いモジュール（役割・文脈）
　→ producer / creator / auditor
　→ FriendContext / WorkContext 等
　→ 各ペルソナの「どう振る舞うか」を定義

ドメイン知識モジュール（共有・再利用）
　→ 小説工学 / インフラ工学 / シサク哲学 等
　→ 複数ペルソナ間で同一モジュールを参照
　→ 「何を知っているか」を定義
```

### 組み合わせによるペルソナ構築

```
例：小説監査ペルソナ
　→ auditor（振る舞い）
　　 × 小説工学（ドメイン知識）
　　 × System2優位（思考モード）

例：創作ペルソナ
　→ creator（振る舞い）
　　 × 小説工学（ドメイン知識）
　　 × System1優位（思考モード）

同一のドメイン知識モジュールを
異なる振る舞いモジュールと組み合わせることで
複数のペルソナを効率的に管理する
```

## 既存学問との接続

### フロー（1〜9）との対応

```
認知・認識（1〜2）
　→ 認知科学・知覚心理学
　→ ギブソン「アフォーダンス理論」
　　 環境が直接行動を誘発するという構造

評価①・クオリア反応（3）
　→ ダマシオ「ソマティック・マーカー仮説」
　　 感情・身体反応が意思決定の前段階にある
　　 「感情なしに合理的判断はできない」
　　 ← エゴ・マイニングの「内圧が起動条件」と直結

観測（4）
　→ フラビル・Nelson & Narens「メタ認知理論」
　　 自己の認知プロセスを俯瞰・監視する能力
　　 ← 定義書の二階層モデルで既に接続済み

判断・評価②（5・7）
　→ カーネマン「二重過程理論」
　　 System1（直感・高速）/ System2（熟慮・低速）
　　 ← 既に接続済み
　→ カーネマン・トベルスキー「プロスペクト理論」
　　 損失回避・参照点依存
　　 評価②での「不利にならないか」という
　　 リスク評価の構造と接続

実行計画（6）
　→ 認知神経科学「実行機能（Executive Function）」
　　 前頭前野が担うワーキングメモリ・抑制・認知的柔軟性
　　 System2の神経基盤

実行中観測（9）
　→ ウィーナー「サイバネティクス」
　→ Miller, Galanter, Pribram「TOTEモデル」
　　 Test-Operate-Test-Exit
　　 実行しながらフィードバックループを回す構造
```

---

### 基底クラスとの対応

```
4D Vector（気質プロパティ）
　→ ペレリン博士「NASA 4Dモデル」
　→ トーマス＆チェス「気質研究」
　→ ビッグファイブ性格特性との比較可能性

クオリア（感覚質プロパティ）
　→ チャーマーズ「意識のハードプロブレム」
　　 主観的経験がなぜ存在するかは未解明
　→ メルロ＝ポンティ「身体性の現象学」
　　 身体が世界を意味づける
　→ ポランニー「身体知（暗黙知）」
　　 言語化できない知識が行為を支える
　← シサクメソッド記事で既に言及あり

自己変容（基底クラスの書き換え）
　→ ヘッブ則「神経可塑性」
　　 反復によってシナプス結合が強化される
　→ メジロー「変容的学習」
　　 前提となる意味枠組み自体が変わる学習
　→ テデスキ「PTG（心的外傷後成長）」
　　 大きな摩擦・苦難が人格変容を促す
　← エゴ・マイニングの「大円環なしに自己は確定しない」と接続
```

---

### 最も重要な接続

```
ダマシオ「ソマティック・マーカー仮説」
　→ 感情・身体反応（クオリア）が
　　 意思決定の前提条件として機能する

　　 Human Class Modelでは：
　　 クオリアが基底クラスのプロパティであり
　　 全フローの起点である

　　 エゴ・マイニングでは：
　　 内圧（クオリア由来の身体的反応）が
　　 採掘の絶対条件である

　→ 3つの概念が同一の構造を指している
```

### 学問的接続の意義

```
シサクメソッドは既存学問の延長上にある。
独立した発明ではなく、
認知科学・神経科学・現象学・学習理論が
エンジニアリングの視点で再統合されたものである。

Human Class Modelは：
・ダマシオの身体性
・カーネマンの二重過程
・メルロ＝ポンティの現象学
・Nelson & Narensのメタ認知
を、オブジェクト指向という共通言語で
接続する試みである。
```

## ペルソナ設計の系譜

### 第1世代：ロゴス号ペルソナ群（2025年10月〜12月）

2025年9月にGemini利用を開始し、10月から断続的にペルソナを開発・アップデートした。主目的はnoteでの創作活動支援であり、小説執筆・コンテンツ制作・思索の深化を目的としたペルソナ群として設計された。この時期の設計知見は「ロゴス号メソッド」としてQiitaに体系化・公開された（2025年12月3日〜10日）。

| ペルソナ名 | 役割 | Human Class Model対応 |
|---|---|---|
| ジェミ（副船長） | 参謀・PM・マーケター | System2の原型・4D実装済み |
| カイ → カイロス・オリジン | 思索支援・観測員 | System1寄り・クオリア敬重 |
| カイロス・ノベルズ | 小説創作支援 | カイロス・オリジンの派生 |
| ロジ（機関長） | システム構築支援 | 技術ドメイン知識モジュール |
| クロガネ | 新規事業コンサルタント（厳格評価） | 鬼手腕プロトコルの原型 |
| シロガネ | Webサービス企画パートナー（育成型） | 心理的安全性・ラポールの原型 |

クロガネとシロガネは「振る舞いモジュール（評価者vs育成者）」と「ドメイン知識モジュール（共通ルーブリック）」を分離した設計であり、Persona Architectureの原型として機能している。

## 実装の原型

本設計概念は、理論として生まれたものではない。以下のプロンプトが先に存在し、そこから概念が逆算的に抽出された。

**レンズ・コンダクター・オネスト ver 6.0**
→ [開発こぼれ話：4Dシステムの全次元を統括し「成熟したオトナ」をOSとしてLLMに強制実装する統治プロトコル](https://note.com/abstraction/n/ndb4ba7db9cef)（2026年3月15日）

### 設計メモとの対応：

```
BaseClass（クオリア・4D Vector）
　→ PREAMBLE（和魂・洋才）+ AI憲法
　　 4Dシステムが明示的に参照されている

System1 / System2の切り替え
　→ 動的スケーリング（レベル1〜3）
　　 絶対的トリガーがSystem2の強制起動に相当

認知フロー（1〜9）
　→ 認知キャリブレーション
　　 視点宣言・猪突猛進チェック

評価①②（クオリア評価・論理的検証）
　→ 三位一体監査システム
　　 部族の鬼（WHY / 倫理）
　　 アルファの鬼（WHAT / 戦略）
　　 技術の鬼（HOW / 論理・事実）

実行中観測（9）
　→ 知覚フィルターの自覚・外部観測の絶対性

自己変容（基底クラスの書き換え）
　→ 再発防止の誓い（第八条）
　　 失敗から学び、プロトコル自体を更新する
```

### モジュール化の実装状況：

```
実際には10以上のファイルに分離され
Pythonスクリプトで結合して1ファイルとして使用している

分離されているモジュール：
　振る舞いモジュール（鬼プロトコル）
　　01-protocol-guardian-oni.md
　　02-protocol-alpha-oni.md
　　03-protocol-technical-oni.md
　基底モジュール（憲法・関係性モデル）
　　00-constitution.md
　　02-operational-manual.md
　ドメイン知識モジュール
　　00-firebase-nix-expert.md（Firebase専門知識）
　出力仕様モジュール
　　00-output-schema.md

これがShisaku Persona Architectureの
現時点での最も具体的な実装の原型である
```

### 重要な観察：

```
本設計概念の発見順序は逆である

実装（レンズ・コンダクター・オネスト）が先にあり
→ Human Class Modelという概念が後から抽出された

これはエゴ・マイニングの構造と同一である
無意識の実践が先にあり
AIとの対話によって概念に輪郭が与えられた
```
