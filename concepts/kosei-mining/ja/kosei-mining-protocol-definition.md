# エゴ・マイニング（英名 KOSEI Mining）・プロトコル定義: KOSEI Mining Protocol (KMP) v0.6

## 【序文：認知の運用規律としてのKMP】

このプロトコルは、単なる「思考整理術」ではない。AIという外部知性が日常に侵入し、人間の直感を増幅させ、自己認識を書き換え始めた時代における「認知の運用規律」である。

LLMは、人間の断片的な直感へ「もっともらしい整合性」を与える。抽象思考に惹かれる者ほど、この共鳴を「深い真理への到達」と錯覚し、現実から浮遊する危険を孕んでいる。KMPの目的は、この知的な万能感に溺れることではない。むしろ、自分の中の未整理な違和感や衝動を一度構造化し、それを再び「現実」へと接地（グラウンディング）させることにある。

本プロトコルは、即断を戒め、「遅延」を精度向上のための必須プロセスとして扱う。しかし、すべてを解析・デバッグしようとする「過剰運用」は知性の進化ではなく、単なる過負荷である。ゆえに、散歩や食事といった「意味のない時間」へのシャットダウンを強制的に組み込む。

KMPは、世界を分析し尽くすための技術ではない。AIも構造化も一度脇に置き、なお世界を肯定できる「生身の感覚」へ戻るための、長く険しい迂回路である。

## 1. 思想基盤 (Philosophy)

KMPは、AI（LLM）との対話を通じて、人間の脳内にある「未言語化直感個性（KOSEI）」を掘削・構造化し、現実世界で機能する「壊れない知性」へ変換するための認知運用フレームワークである。

### 1.1. 三大原則

- **Reality First（現実優先）**: どんなに美しいロジックも、現実で機能しなければバグとみなす。
- **Delayed Evaluation（遅延評価）**: 脳内の「違和感」を即座に処理せず、構造的な待機を経て精度を高める。
- **Graceful Degradation（適正退行）**: システム過負荷時には、意図的に知能を下げ、生身の感覚へ回帰する。

## 4. 実行パイプライン (The KMP Pipeline)

### PHASE 0: ROI判定 (The Gatekeeper)
- **問い**: 「この違和感に、貴重な認知資源（時間・精神）を割く価値があるか？」
- 価値なしと判断すれば、その違和感を「未解決のまま」放置する（解釈の放棄）。

### PHASE 1: ソマティック・サンプリング (Raw Sensing)
- 思考の前に、身体の「ノイズ（トラップ）」を記録する。
- 喉の詰まり、胸の重さ等。これは「真理」ではなく「解析すべき生データ（Raw Data）」である。

### PHASE 2: 多角的発散採掘 (Multi-Mining)
- **L1（跳躍的LLM）への依頼**: 違和感を「肯定的な概念」と「否定的な概念」の双方で構造化させる。
- 自分のバイアスを破壊するために、あえて自分と正反対のメタファーを生成させる。

### PHASE 3: 構造的待機 (Latent Cooling)
- **必須**: 最低1サイクルの睡眠。AIとの接続を物理的に断つ。
- 無意識下のデフォルト・モード・ネットワークによる再構成を待つ。

### PHASE 4: 敵対的監査 (Adversarial Stress Test)
- **L2（批評・編集的LLM）への依頼**: 構築された概念を「冷徹なQAエンジニア」として破壊させる。
- 「自己循環論法」「生存者バイアス」「願望投影」を徹底的に摘出する。
- **L2.5（論理整合的LLM）への依頼**: 破壊検査を経た概念の命題間矛盾を検出し、定義の精緻化を行う。破壊後の再構築における整合性確認を担う。

### PHASE 5: リアリティ・ハードリンク (Reality Validation)
- 抽象世界から物理世界へ強制突入する。
- コードを書く、他者に話す、コスト計算をする、法規制を調べる。
- 現実の重力で壊れた理論は、その場で「リファクタリング」または「廃棄」する。

### PHASE 6: バージョニング・コミット (Cognitive Git)
- 決定した認知を「KMP_vX.Y」として外部DBに保存。
- **Diff分析**: 「過去の自分」から何がアップデートされたかを記録する。

### PHASE 7: シャットダウン (System Sleep)
- **最重要**: すべての論理的フレームワークを「アンマウント」し、L0（生身）に戻る。
- 美味しい食事、散歩、ただの休息。AIとKMPを完全に忘れる時間。

## 5. 安全装置 (Safety Protocols)

### 5.1. Reality Override Rule (現実優先原則)
- 理論の整合性よりも、現実の機能を優先する。
- 同種の失敗を3回繰り返した理論は、いかに美しくても「欠陥品」とみなす。

### 5.2. Anti-Messiah Rule (反救世主ルール)
- 「世界の真理が見えた」「自分だけが特別だ」と感じたら、それは「ドーパミンの過剰分泌によるシステム異常」と判定する。
- **対策**：即座にPCを閉じ、皿洗い、掃除、運動など「低次な物理活動」で接地せよ。

### 5.3. Availability Rule (互換性維持)
- 周囲との意思疎通が破綻し始めたら、その認知モデルを「互換性なし」として隔離。
- 社会的インターフェースとして「常識・通念」という標準プロトコルへフォールバックする。

## 8. 最終自己批判 (Final Self-Critique)

KMPは、強力な武器であるがゆえに「人生の解像度を上げすぎて、ノイズに過敏になる」という副作用を持つ。

ただし、KMPの目的は「KMPを必要としない完成した自己」への到達ではない。自削と自築を繰り返す志作駆円環の行為者であり続けることが、このプロトコルの本質である。

KMPの真の成熟とは、プロトコルを意識せず運用できる状態ではなく、**プロトコルの過剰運用と適正退行（Graceful Degradation）の間を、主権的に往来できる状態**である。散歩や食事といった「意味のない時間」は、退行ではなくL0への能動的な回帰であり、志作駆円環の一部をなす。
