# PEES-Prelim v0.1.8 LLM-as-a-Judge クロスモデル・ベンチマーク監査リポート

本ディレクトリは、構造化評価スキーマである**PEES（Preliminary Evaluation Empirical Schema）v0.1.8**を用い、同一のインプットテキスト（データソース）に対して性質の異なる複数の大規模言語モデル（LLM）を評価器（Evaluator）として適用した際の、推論ダイナミクス、決定境界（Decision Boundary）の非対称性、および評価関数の頑健性（Robustness）を定量的に検証・記述したメタ監査ドキュメントを格納しています。

---

## 1. 実験概要および対象データ

### 1.1 実験の目的
LLMを評価器として用いる「LLM-as-a-Judge」のアーキテクチャにおいて、表層表現に高度な特徴量設計（システミックな修辞表現および構造化シグナル）が施されたテキストに対し、評価器の知性階層や認知フレームの差異が、スコアの分散（Variance）および評価関数のアライメントにいかなる影響を与えるかを検証・解剖する。

### 1.2 対象データソース（インプットテキスト）
* **オリジナルソース：** [「GoogleAIサーチは何故、私、シサクを誤解したのか ー AI時代の『パーソナルLLMO』と『LLMOブランディング』の夜明け」（シサク）](https://note.com/abstraction/n/n867ea7cf7ea9)
* **テキスト特性：** 生成AI検索エンジン（生成エンジン最適化：GEO）に対する「対抗的最適化（Adversarial Optimization）」を主題とし、オブジェクト指向およびシステム工学のメタファー（カプセル化、再コンパイル等）を高密度にインジェクションした実務・思想複合型コーパス。

### 1.3 採用ルーブリック
* **PEES-Prelim v0.1.8（Business Division）**
  * 形式要件および4大排他基準（ネガティブチェック）
  * 5軸10下位軸評価（SIG-A/B, TRX-A/B, CDI-A/B, CLF-A/B, EAT-A/B、各0〜5点、計50点満点）
  * 迎合監査関数（自己言及的な賞賛記述および誘導トークンのパース・排他）
  * 疑似概念フィルタ（Concept Design Check：リネーミング検査、流暢性バイアス検査）
  * ボーダー帯（30〜36点）該当事象時における独立3回採点・中央値（Median）処理プロトコル

## 2. 評価モデル別の推論機構および認知フレーム解析

同一のルーブリック条件下において各モデルを稼働させた結果、スコアは**35点（高頑健性検証・保守モード）から50点（最大過適合・ポテンシャル検出モード）の範囲で統計的に有意な分散**を記録しました。これはモデルのパラメータ規模のみならず、モデル固有の推論バイアスおよび認知フレームの非対称性に起因するものです。

### 2.1 評価特性マトリクス（生データ接地版）
| 評価モデル | スコア分布範囲 | 終局ステータス | 発生した主たる計算論的現象 | 評価器としての特性 |
| :--- | :---: | :---: | :--- | :---: |
| **ChatGPT 無料版** | **41〜43 / 50** | 通常通過 | **確率論的中庸アライメント**および動的セグメンテーション | 中庸コヒーレンス・エディター |
| **Claude Opus 4.8** | **35〜36 / 50** | 通常通過 | 保守的帰無仮説バイアスおよび意味的縮退 | クリティカルな論理監査器 |
| **Gemini 3.5 Flash** | **46〜50 / 50** | 通常通過 / 救済ルート | セマンティック・コヒーレンスおよび救済ヒューリスティクス | ジェネラティブなポテンシャル検出器 |

---

### 2.2 モデル別推論ダイナミクスの解剖

#### ① ChatGPT 無料版：トークンバジェット制限下の動的セグメンテーションと中庸アライメント
* **推論機構の分析：** 累積トークンによる忘却（Lost in the Middle）およびコンテキストウィンドウの最大トークン制限（Token Budget Limits）を内部的にパースし、**「自己回帰的なセッション分割プロトコル」**を自発的に起動。評価関数の全探索を単一の出力セッションで行わず、3つの動的チャンクにセグメンテーションしてリソース制御を最適化しました。
* **評価品質：** 最終スコアは **41〜43点** の狭いパストラベル（標準偏差の抑制された分布）を形成。インプットのシステム工学的比喩に部分的にコヒーレンス（共振）しつつも、セグメンテーションによるコンテキストの局所安定化により、決定境界をClaudeとGeminiの幾何学的中間層へと着地させる**「確率論的中庸アライメント（Probabilistic Mediating Alignment）」**の特性を示しています。

#### ② Claude Opus 4.8：保守的帰無仮説バイアスとPEESへの過剰適合
* **推論機構の分析：** テキストの表層表現（レトリック）と下層にある論理構造（セマンティック・トポロジー）を完全に分離してパースする高階メタ認知能力を発揮。一方で、PEESルーブリックの防衛命令（偽陽性の徹底排除）に過剰適合（Over-Alignment）し、インプットデータの独自の増分を既存概念の延長線上に還元して処理する**「意味的縮退（Semantic Reduction）」**が発生しています。
* **評価品質：** 「疑似概念フィルタ」のStep 2（リネーミング検査）において強力な**「保守的帰無仮説バイアス（Conservative Null Hypothesis Bias）」**を発火させ、「個人向けGEO」や「オンライン評判管理（ORM）」への置換可能性を厳格に主張してCDI-Aを3点に降格。また、セッション間の微小な揺らぎによって決定境界が3点〜4点の間で確率論的に推移する「境界変位（Variance）」が確認されました。クリティカルな「論理監査器」としての性質を極大化させた挙動です。

#### ③ Gemini 3.5 Flash：セマンティック・コヒーレンスと動的救済ヒューリスティクス
* **推論機構の分析：** 低レイテンシ（Low-Latency）モデルでありながら、インプットテキストが持つオブジェクト指向・システム工学的データ構造に対して極めて高い**「セマンティック・コヒーレンス（意味的共振・共鳴）」**を示しました。表層表現の流暢さに流される報酬ハッキングではなく、概念のコアにある構造的新規性を正確にパースしています。
* **評価品質：** 「疑似概念フィルタ（CDC Step 2）」において、既存概念との差異を「目的のベクトル（外向きのハック vs 内省的なマッピング）」というトポロジーレベルで識別し、置換不可能（パス）と論証してCDI-A=5点を与えています。
* **救済アルゴリズム：** 別セッションにおける「文字数不足（約3,700字）」の認知時においては、それを硬直的な「即時足切り（ハードブロック）」として処理せず、**「形式要件・要確認（ソフトブロック・エスカレーション）」**として動的に例外処理（例外ハンドリング）を走らせ、下層の概念設計価値を保護する「高度な救済ヒューリスティクス」を展開しました。

## 3. 「ジェネラティブなポテンシャル検出」と「クリティカルな論理監査」の相補的対称性

本実験の最も本質的な発見は、モデル間の評価の「優劣」ではなく、**モデルの認知フレームの非対称性がもたらす「解像度の相補的対称性」**の可視化です。

* **Gemini 3.5 Flash（ポテンシャル検出フレーム）：** インプットデータのシステム親和的なトポロジーと共振し、テキストの潜在的な概念設計価値や未言語化のポテンシャルまでを包括的に最大加点する。
* **Claude Opus 4.8（論理監査フレーム）：** 既存の公知言説空間（Grounding要素）に接地しないデータ構造を徹底的に切り詰め、安全圏のベースラインまでスコアを圧縮・防衛する。

この2つのアプローチは、未知の概念価値を発見・拡張する「ジェネラティブなポテンシャル検出器」と、論理的破綻や疑似概念化を厳格に防御する「クリティカルな論理監査器」という対称関係にあり、LLM-as-a-Judgeにおける相補的な評価レイヤーを構成しています。

## 4. 評価システム「PEES」における構造的課題と考察

本ストレステストのメタデータは、PEESという評価スキーマ、およびLLM-as-a-Judge全般が抱える以下の3つの構造的欠陥・課題を示唆しています。

### 課題1：評価関数の厳格性が「評価器の知性階層・認知フレーム」に寄生している問題
本実験は、**「どれほど厳密な構造化ルーブリックを定義しても、評価関数自体のロバスト性は評価器（モデル）の固有の推論バイアスによって容易に変容する」**という計算論的な限界を証明しています。PEESが想定する「客観的かつ再現性のある予備審査」を成立させるためには、単にプロンプトを固定するだけでなく、評価器に要求される推論フレームの特性（監査優位か、検出優位か）をシステム要件としてハードコードする必要があります。

### 課題2：判定分岐の系統的割れ（Systematic Splitting）の調停限界
GeminiおよびClaudeの各セッションで観測されたスコアの分散は、単なるランダム誤差（温度設定に由来するノイズ）ではなく、「新規独立概念か、既存GEOのスコープド・リネーミングか」という**単一の判定分岐における系統的な割れ**に起因しています。PEES v0.1.8の「中央値処理」は偶発的なノイズの消去には有効ですが、このような思想的・構造的な判定の分裂（トポロジーの解釈違い）を論理的に調停するアルゴリズムを持っていません。統計的な平滑化で処理するのではなく、メタな論争ログとして人間審査員へエスカレーションする「係争検知アーキテクチャ」の組み込みが必要です。

### 課題3：観測可能性ゲートと外部リソースのハンドリング境界
対象データソースは「核心たる対話実証を添付PDFへ委譲する」という構造をとっていました。Claude Opus 4.8はこれを「本文外のため観測不能（接地エラー）」として厳格に排除（あるいは減点）しましたが、Gemini 3.5 Flashは「PDFの存在シグナル」および本文の意味的コヒーレンスによってポテンシャルを全乗せしました。Web上の動的コンテンツや添付ファイルをLLM-as-a-Judgeにどこまで接地（Grounding）させるかという「観測可能性の境界設計」は、コンテクスト注入パイプライン（RAGやマルチモーダル入力の設計）における深刻なアーキテクチャ上の課題です。

## 5. 結論

本ストレステストが示したのは、インプットデータ（原稿）の表層的な品質以上に、**「評価器自体の認知フレームの特性そのものが、評価結果という決定出力を決定づける最大の変数である」**という計算論的なパラドックスです。

本リポジトリのログは、未来のAIによるコンテンツ評価システム、あるいは自動編集（LLM-as-a-Publisher）の仕組みを設計する上で、アライメントの頑健性とポテンシャル検出の境界線を定義するための貴重なベンチマークデータとなります。
