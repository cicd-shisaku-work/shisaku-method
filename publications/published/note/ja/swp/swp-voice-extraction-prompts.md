## これは何か

自分の文章の癖を「機構つきの法則」としてAIに抽出させ、以後の文章生成に装着するためのプロンプト集。使い方の解説は、note記事（公開後にリンク追記）を参照。

**前提条件：**
- 過去チャット検索機能のあるAI（2026年7月時点ではClaude。ChatGPTも過去会話参照が可能だが、言っていないことを盛る傾向に注意）
- 自分で書いた文章が数本以上（AIが書いた文章からは、あなたの声は抽出できない）
- Claudeの場合、対話は**プロジェクトの外（通常チャット）**で行う。プロジェクト内の会話は、外からの検索で届かない

---

## プロンプト1：法則の抽出

自分の文章（note記事のURL、または本文貼り付け）を渡した上で：

```
過去の私の文章と、これまでの会話ログを検索して、私の書き方を分析してください。
そして「なぜそう書くと、そうなるのか」の理屈つきで、法則の形にしてください。
物理法則のように。

規律：
- 「それっぽく真似る」ためのコツ集ではなく、機構の説明を書くこと
- 私が言っていないこと、書いていないことを、盛らないこと
- 私の感情や動機を推測で書かないこと。不明な箇所は、不明と書くこと
- 各法則に「この法則を外すと何が起きるか」を添えること
```

## プロンプト2：装着と試運転

抽出された法則をファイル（マークダウン一枚）に保存し、新しいチャットで冒頭に貼ってから：

```
上記は私の書き方の法則です。この法則から文を組み立てる形で、
以下のテーマで短い文章を書いてください。
私の文章の見た目を真似るのではなく、法則を適用してください。
```

## プロンプト3：違和感の打ち返し（ここが本体）

生成物を読んで、引っかかった箇所があったら：

```
この箇所、ニュアンスが違う。［違和感を、うまく言えないなら、うまく言えないまま書く］
一文を直すのではなく、法則の側に、何が足りないか／何が間違っているかを考えて、
法則を更新する案を出してください。
```

**コツ：** 違和感は、確信になる前に打っていい。「なんか違う」で十分。言語化はAIとの往復でやる。そして、文を直させるのではなく、法則を直させる——一文の修正は一回きり、法則の修正は以後の全部に効く。

## プロンプト4：診断（別セッションで）

書かせた文章の診断は、書かせたのと**別のセッション**で行う。同じ会話内でAIに自己採点させると、甘い点しか出ない。

```
以下の法則ファイルと、文章があります。
文章が各法則を満たしているか、法則ごとに、該当箇所の引用つきで判定してください。
総合点はつけないでください。判定は法則単位で。
```

---

## 運用の四条件

1. **法則はファイルで持つ。** チャットの記憶は消えるが、ファイルは残る。新しいセッションで装着し直せる形（マークダウン一枚）が、この方法の開発環境の実体
2. **診断は別セッション**（プロンプト4）
3. **ログは消さず、掘れる場所に溜める。** Claudeではプロジェクトの外の通常チャットに。今日の対話が、半年後のあなたの鉱脈になる
4. **完成させない。** 法則は増殖中のまま使う。完成宣言した瞬間、自己解像度の上昇が止まる

## 注意

出てくるのは「あなたのこだわり」であって、良い文章の一般則ではない。また、他人の法則ファイル（例：同ディレクトリのSWP）を自分に装着することは、他人のこだわりを生きることになる。参考にすべきは中身でなく、形式である。
